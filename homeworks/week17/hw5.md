## 這週學了一大堆以前搞不懂的東西，你有變得更懂了嗎？請寫下你的心得。
這一週學習到了不少東西，稍微條列了一下：scope、hoisting、closure、prototype、this、Event Loop 等等。
原本在這週之前，都還看不懂同學分享的面試題目，完成 17 週了之後，才終於豁然開朗，難怪當初是建議 17 週以後可以出去面試。

這週的教材，依據導讀影片介紹的順序：
1. (影片) 程式導師實驗計畫第二期：Week9-2 JavaScript 執行原理
2. (文章) 該來理解 JavaScript 的原型鍊了
3. (文章) 深入探討 JavaScript 中的參數傳遞：call by value 還是 reference？
4. (文章) 我知道你懂 hoisting，可是你了解到多深？
5. (文章) 所有的函式都是閉包：談 JS 中的作用域與 Closure
6. (文章) 淺談 JavaScript 頭號難題 this：絕對不完整，但保證好懂
7. (課程) [JS201] 進階 JavaScript
8. (影片) 程式導師實驗計畫：Lesson 9-2 之資料結構、Cache 與 Event Loop 
9. (影片) What the heck is the event loop anyway? | Philip Roberts | JSConf EU
我原本是照順序來學，剛開始學有點消化不良，似懂非懂，後來跳著先看了 [JS201] 的影片，一節一節的去研讀，才慢慢有點開竅。
接著由 [JS201] 出發，回頭看相關的影片部分和文章，學習就順暢了起來。
我覺得 [JS201] 很棒的地方在於，把一個大觀念細分成很多小觀念，逐步的建立起對於名詞的認識。這樣的做法很像是寫程式，把一個感覺很難的功能，細分成很多步驟，去逐一完成每一個小函式，最終完成整個作品的概念。
而且因為每一節講的東西很單純，標題也適切，所以當覺得不太確定想回頭重看時，很容易就能找到重點片段去複習，這一點幫助很大也很重要，裡面研讀 ECMA 的章節、模擬 JS 引擎的章節、作用域陷阱等等，我都回頭重新看了幾次。鋰學院對於課綱的安排很有一套。
個人認為經過這週，在 scope, hoisting, this, Event Loop 方面已經有足夠基本的了解，也有自信解答相關的基礎題目，但對於 closure，prototype, 原型鍊等等，還處於似懂非懂的狀態，可能程式寫得不夠，時候未到，沒有切身體驗「這個問題會造成困擾」，所以認識得比較粗淺，不過有了一些先備知識後，以後遇到問題時，至少還能有個方向去 google。
另一個角度來說，這週還有個體驗就是：呼籲各位工程師：乖乖用 let 取代 var 宣告變數吧！
